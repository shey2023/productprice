# תקציר קוד — Shey מחשבון הצעות מחיר לתכשיטים

## סקירה כללית
אפליקציית PWA חד-עמודית (SPA) בעברית, בנויה מ-HTML/CSS/JS טהור ללא תלויות חיצוניות. המטרה: לסייע לצורפים ולבעלי עסקי תכשיטים ליצור הצעות מחיר מקצועיות — חישוב עלויות מתכת, יהלומים, אבני חן ועבודה, עם ייצוא PDF ללקוח (ללא פירוט עלויות) ו-PDF פנימי (עם כל הפירוט). כל הנתונים נשמרים בדפדפן בלבד ב-`localStorage`.

---

## קבצים עיקריים

- **`index.html`** — מבנה ה-UI המלא: כרטיסיות קלט (סוג תכשיט, מתכת, משקל, אבנים, עלויות, רווח, הערות), לוח תוצאות עם כפתורי PDF, היסטוריה, ו-overlay ל-PDF. RTL, שפה `he`.
- **`app.js`** — כל הלוגיקה: חישובים, עדכון UI, ייצוא PDF, שמירה/טעינה מ-localStorage, PWA manifest, event listeners.
- **`styles.css`** — עיצוב responsive בנושא ורוד/זהב, CSS variables, תמיכה במובייל.

---

## זרימות משתמש עיקריות

1. **הזנת נתונים** — משתמש בוחר סוג תכשיט ← מתכת + מחיר לגרם ← משקל (ידני או חישוב לפי מידות לטבעת) ← אבנים (יהלומים / אבני חן) ← עלויות עבודה ← אחוז רווח.
2. **חישוב אוטומטי** — כל שינוי בשדה מפעיל `scheduleCalc()` (debounce 120ms) שמחשב ומציג תוצאות בזמן אמת.
3. **שמירה** — לחיצה על "שמור ★" שומרת את כל הנתונים ב-`localStorage` (עד 50 הצעות).
4. **טעינה** — לחיצה על "טען" בהיסטוריה משחזרת את כל שדות הטופס.
5. **ייצוא PDF** — שני מצבים (ראה להלן), פתיחת overlay עם CSS print.

---

## מודל הנתונים / localStorage

- **מפתח**: `jc_hist_v2`
- **מבנה**: מערך JSON של עד 50 פריטים. כל פריט:
  ```json
  {
    "id": "x...",
    "date": "תאריך ושעה",
    "client": "שם לקוח",
    "jewLabel": "טבעת",
    "metalLabel": "זהב 14K",
    "total": 4500,
    "note": "הערה קצרה",
    "payload": { /* כל שדות הטופס + diamondSpecs + dynCosts + gemStones */ }
  }
  ```
- **`diamondSpecs`** (בתוך `payload`):
  ```json
  {
    "uni": { "clarity": "VS1", "color": "E", "gia": "", "showInClient": { "clarity": true, "color": false, "gia": false } },
    "g1":  { ... },
    "g2":  { ... }
  }
  ```
  — מאפשר פירוט נפרד ליהלום אחיד (`uni`) ולשתי קבוצות (`g1`, `g2`).

## מצבי ייצוא PDF

| מצב | תוכן |
|-----|------|
| **PDF ללקוח** (`client`) | סוג תכשיט, מתכת, ומאפייני יהלומים שסומנו "הצג ללקוח" — **ללא** פירוט עלויות. מחיר סופי בלבד. |
| **PDF פנימי** (`internal`) | פירוט מלא: מתכת, יהלומים, אבני חן, עלויות עבודה, רווח, הערות פנימיות. מאפייני יהלומים שלא סומנו להצגה מסומנים "(פנימי בלבד)". |

הלוגיקה נמצאת ב-`openPdf(mode)` ב-`app.js`. ה-overlay מודפס דרך CSS (`@media print`).

---

## פירוט יהלומים (Diamond Specs)

- **שדות**: נקיון (Clarity — FL עד I3), צבע (Color — D עד K-Z), מספר תעודת GIA (טקסט חופשי).
- **שני מצבים**: אחיד (`uni`) — ספציפיקציה אחת לכל היהלומים; מרובה (`multi`) — ספציפיקציה נפרדת לקבוצה 1 (מרכזית) וקבוצה 2 (צדדיות).
- **Checkboxes** לכל שדה: "הצג בהצעה ללקוח" — שולט מה נכנס ל-PDF ללקוח.
- רשימות dropdown מאוכלסות ב-runtime מ-`CLARITY_OPTIONS` ו-`COLOR_OPTIONS`.

---

## PWA / Manifest

- ה-manifest נוצר דינמית ב-JS (`Blob` + `URL.createObjectURL`) כדי לא לדרוש קובץ נפרד.
- הגדרות: `display: standalone`, `theme_color: #D4537E`, `lang: he`, `dir: rtl`.
- האירוע `beforeinstallprompt` נלכד ונשמר ב-`deferredPrompt`, אך אין כרגע כפתור "הוסף למסך הבית" בממשק — הבסיס קיים לשימוש עתידי.
- **אין Service Worker** — האפליקציה אינה עובדת offline.
